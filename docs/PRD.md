## 1. 背景与目标（Why）

### 1.1 背景
徒步、自行车骑行等户外活动常进入移动网络覆盖弱或完全无信号的区域（山区、郊野、隧道、偏远公路）。现有对讲机或网络通话应用在无网环境下失效，导致小队成员之间无法实时沟通、掉队难以发现、突发情况难以协调。

### 1.2 产品目标
打造一款**开源 Android 应用**，让 2-5 人小型户外团队在不依赖任何基站/互联网的情况下，通过**一台手机开启热点、其他手机接入该热点**的方式组成局域网，实现：

- 实时全双工语音群聊；
- 队友相对位置共享；
- 网络可扩展：在单跳覆盖范围内可支持更多节点接入。

> 说明：MVP 采用**星型单跳**拓扑。多跳中继、去中心化 mesh 已明确从 v1.0 需求范围移除，避免对手机 AP-STA 并发能力的依赖。

### 1.3 成功指标
- 小队成员在无移动网络环境下，仍能在热点覆盖范围内保持连续语音沟通；
- 应用在 GitHub 开源，社区可复用/审计/贡献。

## 2. 用户画像与场景（Who + Where）

### 2.1 核心用户画像

| 维度 | 描述 |
|------|------|
| 用户类型 | 徒步爱好者、自行车骑行小队、越野跑团队 |
| 团队规模 | 2-5 人为主 |
| 设备 | Android 12+ 手机，佩戴有线麦克风耳机或蓝牙耳机 |
| 技术素养 | 普通用户，可接受首次使用时由一人开启热点并选择 Group Owner |
| 使用环境 | 山区、郊野、城市边缘等无移动网络或弱网环境，队员间距在热点覆盖范围内 |

### 2.2 典型场景

**场景 A：山地徒步小队**
> 小明和 4 位朋友进山徒步，山路起伏导致前后队员拉开 30-80 米，且无手机信号。小明在一加 11 上选择 Group Owner 并开启热点，其他队员选择 Client 接入该热点。大家打开 OffGrid 后即可在同一局域网内语音群聊。

**场景 B：公路骑行小队**
> 3 人骑行小队沿公路骑行，需要前后照应、提醒路况、协调休息点。由领队手机作为热点，其他队员接入，通过蓝牙耳机连接手机实现免提语音群聊，同时可在屏幕上看到队友相对位置。

**场景 C：野外探险备用通讯**
> 进入无信号区域前，团队提前安装并配置好 OffGrid，指定一台设备作为热点，作为紧急情况下的备用沟通手段。

## 3. 功能需求（What）

### 3.1 核心功能

| 编号 | 功能 | 说明 | 优先级 |
|------|------|------|--------|
| F01 | 局域网语音通话 | 多台手机接入同一热点后，实现实时语音群聊 | Must |
| F02 | 节点发现与接入 | 应用启动后，Client 端可发现当前 Group Owner 并一键接入 | Must |
| F03 | 位置共享 | 在应用内显示队友相对方位与距离 | Must |
| F04 | 音频输入输出 | 支持有线麦克风耳机与蓝牙耳机作为音频输入/输出设备 | Must |
| F05 | 开源发布 | 代码以开源协议发布在 GitHub，附带构建说明与贡献指南 | Must |

> 注：原 F02「多跳中继」已删除。

### 3.2 网络配置功能

| 编号 | 功能 | 说明 | 优先级 |
|------|------|------|--------|
| F06 | 手动选择角色 | 用户可在 Settings → Connection 中选择 **Group Owner（热点）** 或 **Client（接入）** | Must |
| F07 | 网络状态显示 | 显示当前角色、连接状态、已接入节点数、热点信号强度 | Should |
| F08 | 自动重连 | Client 短暂离开覆盖范围后回到网络中时，自动恢复连接 | Should |
| F09 | 热点切换提示 | 当 Group Owner 电量低或离开网络时，提示用户重新选择 Group Owner | Could |

### 3.3 增强功能（后续迭代）

| 编号 | 功能 | 说明 | 优先级 |
|------|------|------|--------|
| F10 | 离线文字消息 | 在语音不便时发送简短文字 | Could |
| F11 | 语音回放/录音 | 重要信息可本地录音或回放 | Could |
| F12 | 离线地图加载 | 预加载离线地图作为位置共享底图 | Could |

## 4. 非功能需求

### 4.1 性能

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 热点覆盖半径 | ≥ 30m（开阔地，受手机天线与地形影响） | 所有 Client 必须在 Group Owner 热点覆盖范围内 |
| 端到端语音延迟 | 单跳 < 300ms | 热点 → Client 或 Client → 热点 → Client |
| 语音质量 | 在低速率编码下保持可懂度（MOS ≥ 3.0） | 户外抗风噪优先 |
| 支持节点数 | 2-5 台手机稳定运行 | 受手机热点最大连接数限制 |

> 注：原「支持的跳数：至少 3-5 跳」已删除。

### 4.2 兼容性与平台

- 最低系统版本：Android 12（API 31）；
- 目标设备：主流 Android 手机，支持 Wi-Fi Direct Group Owner / Client 或 SoftAP；
- 音频外设：有线耳机（带麦克风）、蓝牙耳机（HFP/A2DP）；
- 开源协议：MIT。

### 4.3 可靠性与可用性

- 应用在后台持续运行，锁屏后仍可接收语音；
- Client 进出网络时，邻居表自动更新；
- 提供电量优化白名单引导，防止系统杀后台；
- 首次使用提供引导教程；
- Group Owner 离开网络时给出明确提示，建议用户重新选择热点设备。

### 4.4 安全与隐私

- 所有通信仅在设备间本地完成，不经过任何服务器；
- 位置数据仅在本地局域网内广播，不上传云端；
- 建议对控制信令与语音载荷进行加密（如网络层 ChaCha20-Poly1305），防止未授权设备接入热点后窃听。

### 4.5 电源与续航

- 持续语音通话场景下，每小时耗电 ≤ 15-20%（目标）；
- 不提供独立的「省电模式」开关（已从需求范围删除）；
- 提供屏幕关闭后纯后台语音模式；
- Group Owner 设备耗电会显著高于 Client，需在 UI 中提示用户。

## 7. 边界与排除项

### 7.1 范围内
- Android 12+ 手机端应用；
- 基于一台手机作为热点（Group Owner / SoftAP）、其他手机作为 Client 的星型组网；
- 离线环境下的语音与位置数据交换；
- 开源代码与文档。

### 7.2 范围外（Won't have / 暂不实现）
- iOS 版本（首期）；
- 多跳中继 / 去中心化 mesh；
- 通过互联网/服务器中转的远程通话；
- 脱离手机硬件的独立硬件对讲机；
- 付费服务、账号系统、云端存储；
- 超出手机热点覆盖范围的长距离通信；
- 复杂的动态角色切换与自动负载均衡。

### 7.3 关键假设与风险

| 假设 | 风险 | 缓解措施 |
|------|------|----------|
| 用户可接受由一人开启热点并选择 Group Owner | 配置复杂导致流失 | 提供清晰引导与默认推荐方案 |
| 手机热点在户外可达 ≥ 30m | 实际距离受地形、电量、天线影响 | 实机测试不同场景，设置合理预期 |
| Group Owner 设备电量充足 | 热点设备耗电快，可能提前退出 | UI 提示电量、支持手动切换 Group Owner |
| Android 后台限制不阻断服务 | 系统杀后台导致断连 | 引导用户设置电池白名单，使用前台服务 |

## 8. 用户故事与验收标准

### 8.1 用户故事 1：作为徒步领队，我希望在无网环境下与全队保持语音联系

**验收标准：**
- 启动应用后，指定一台手机作为 Group Owner 开启热点，其余 Client 在 60 秒内完成接入；
- 领队说话，所有队员能听到；
- 任意队员说话，领队能听到；
- 通话过程中延迟 < 300ms。

### 8.2 用户故事 2：作为队员，我希望简单快速地加入领队的网络

**验收标准：**
- 在 Settings → Connection 中选择 Client；
- 点击 Start Call 后扫描并显示当前可用的 Group Owner；
- 一键接入后，Neighbors 列表显示所有在线节点；
- 若当前无可用 Group Owner，给出明确提示。

### 8.3 用户故事 3：作为骑行队员，我希望通过蓝牙耳机免提通话

**验收标准：**
- 应用可正确识别蓝牙耳机麦克风与扬声器；
- 通话过程中可通过耳机按键切换静音或挂断；
- 蓝牙耳机断开时，自动切换回手机扬声器/麦克风。

### 8.4 用户故事 4：作为团队成员，我希望看到队友相对位置

**验收标准：**
- 组网成功后，方位界面显示每个队友的相对方位与距离；
- 位置刷新频率 ≥ 1 次/5 秒；
- 位置数据仅在本地局域网内传输，不上云。

## 9. 跨角色转译

### 9.1 产品语言（Product）
> 做一款“无网也能群聊”的户外通讯工具。核心价值是**不依赖运营商、在热点覆盖范围内实现小队语音+位置共享**。首期聚焦 2-5 人小队，由一人开启热点，其他人接入即可使用。

### 9.2 开发语言（Development）
> 需要实现：
> - Android 应用框架（Kotlin/Java）；
> - 星型局域网层（基于 Wi-Fi Direct Group Owner / Client 或 SoftAP）；
> - 语音采集/播放（AudioRecord/AudioTrack）与低码率编解码（Opus）；
> - 局域网内 UDP 广播/单播发现与语音转发；
> - 位置广播（GPS/北斗 + 本地广播）；
> - 开源仓库与 CI 构建。

### 9.3 测试语言（QA）
> 关键验证：
> - 实机热点组网稳定性（3-5 台设备）；
> - 单跳语音延迟与可懂度；
> - 后台保活与断线重连；
> - Group Owner 切换场景；
> - 不同耳机类型兼容性；
> - 无网环境下的位置同步；
> - 开源构建可复现性。

### 9.4 设计语言（Design）
> - 首页：一键选择角色/接入/通话、节点列表、网络状态；
> - 方位页：队友相对方位与距离；
> - 设置页：角色选择、音频设备、电池白名单引导、热点切换提示；
> - 首次启动引导：权限申请、角色说明、耳机配对提示。

### 9.5 PM 语言（Project Management）
> - 项目周期：3-6 个月；
> - 关键路径：组网方案选型 → 直连语音 Demo → 多 Client 接入 → 位置共享 → 后台优化 → 开源发布；
> - 外部依赖：Android 权限策略、设备兼容性、热点最大连接数；
> - 建议先交付 MVP 验证核心链路，再迭代增强功能。
