## 1. 背景与目标（Why）

### 1.1 背景
徒步、自行车骑行等户外活动常进入移动网络覆盖弱或完全无信号的区域（山区、郊野、隧道、偏远公路）。现有对讲机或网络通话应用在无网环境下失效，导致小队成员之间无法实时沟通、掉队难以发现、突发情况难以协调。

### 1.2 产品目标
打造一款**开源 Android 应用**，让 2-5 人小型户外团队在不依赖任何基站/互联网的情况下，通过手机自组**去中心化网状网络（mesh）**，实现：

- 实时全双工语音群聊；
- 队友相对位置共享；
- 多跳中继：队伍前后端人员可通过中间手机节点接力通信；
- 网络可扩展：在节点数量足够时，理论上支持更多跳数与更广覆盖。

### 1.3 成功指标
- 小队成员在无移动网络环境下，仍能保持连续语音沟通；
- 前后队员距离超出单跳覆盖时，可通过中间节点自动中继；
- 应用在 GitHub 开源，社区可复用/审计/贡献。

## 2. 用户画像与场景（Who + Where）

### 2.1 核心用户画像

| 维度 | 描述 |
|------|------|
| 用户类型 | 徒步爱好者、自行车骑行小队、越野跑团队 |
| 团队规模 | 2-5 人为主，可扩展至 10 人左右 |
| 设备 | Android 12+ 手机，佩戴有线麦克风耳机或蓝牙耳机 |
| 技术素养 | 普通用户，可接受首次使用时进行简单的网络配置 |
| 使用环境 | 山区、郊野、城市边缘等无移动网络或弱网环境 |

### 2.2 典型场景

**场景 A：山地徒步小队**
> 小明和 4 位朋友进山徒步，山路起伏导致前后队员拉开 200-500 米，且无手机信号。大家打开 OffGrid，手机自动组成网状网络，走在最前面的小明可以与最后面的小红通话，中间队员的手机自动中继语音。

**场景 B：公路骑行小队**
> 3 人骑行小队沿公路骑行，需要前后照应、提醒路况、协调休息点。通过蓝牙耳机连接手机，实现免提语音群聊，同时可在屏幕上看到队友相对位置。

**场景 C：野外探险备用通讯**
> 进入无信号区域前，团队提前安装并配置好 OffGrid，作为紧急情况下的备用沟通手段。

## 3. 功能需求（What）

### 3.1 核心功能

| 编号 | 功能 | 说明 | 优先级 |
|------|------|------|--------|
| F01 | 网状语音通话 | 多台手机自组网状网络，实现实时语音群聊 | Must |
| F02 | 多跳中继 | 当发送方与接收方不在直连范围时，自动通过中间节点转发语音包 | Must |
| F03 | 节点发现与组网 | 应用启动后自动发现附近已安装本应用的节点并完成组网 | Must |
| F04 | 位置共享 | 在应用内显示队友相对方位与距离 | Must |
| F05 | 音频输入输出 | 支持有线麦克风耳机与蓝牙耳机作为音频输入/输出设备 | Must |
| F06 | 开源发布 | 代码以开源协议发布在 GitHub，附带构建说明与贡献指南 | Must |

### 3.2 网络配置功能

| 编号 | 功能 | 说明 | 优先级 |
|------|------|------|--------|
| F07 | 手动网络配置 | 用户可手动选择组网方式（如 Wi-Fi Direct、手机热点等） | Should |
| F08 | 网络拓扑可视化 | 显示当前节点连接关系、信号强度、跳数 | Should |
| F09 | 自动重连 | 节点短暂离开覆盖范围后，回到网络中时自动恢复连接 | Should |

### 3.3 增强功能（后续迭代）

| 编号 | 功能 | 说明 | 优先级 |
|------|------|------|--------|
| F10 | 离线文字消息 | 在语音不便时发送简短文字 | Could |
| F11 | 语音回放/录音 | 重要信息可本地录音或回放 | Could |
| F12 | 离线地图加载 | 预加载离线地图作为位置共享底图 | Could |

## 4. 非功能需求

### 4.1 性能

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 单跳有效距离 | ≥ 100m（Wi-Fi Direct，开阔地） | 受硬件与环境影响 |
| 端到端语音延迟 | 单跳 < 300ms；每增加一跳增加 < 200ms | 可感知但可接受 |
| 支持的跳数 | 至少 3-5 跳稳定运行 | 覆盖典型小队前后拉开场景 |
| 语音质量 | 在低速率编码下保持可懂度（MOS ≥ 3.0） | 户外抗风噪优先 |

### 4.2 兼容性与平台

- 最低系统版本：Android 12（API 31）；
- 目标设备：主流 Android 手机，支持 Wi-Fi Direct；
- 音频外设：有线耳机（带麦克风）、蓝牙耳机（HFP/A2DP）；
- 开源协议：MIT。

### 4.3 可靠性与可用性

- 应用在后台持续运行，锁屏后仍可接收语音；
- 节点进出网络时，网络拓扑自动收敛；
- 提供电量优化白名单引导，防止系统杀后台；
- 首次使用提供引导教程。

### 4.4 安全与隐私

- 所有通信仅在设备间本地完成，不经过任何服务器；
- 位置数据仅在网状网络内广播，不上传云端；
- 建议对控制信令与语音载荷进行加密（如 DTLS/SRTP 或 mesh 层加密），防止附近未授权设备窃听。

### 4.5 电源与续航

- 持续语音通话场景下，每小时耗电 ≤ 15-20%（目标）；
- 提供省电模式：降低位置刷新频率、降低语音码率；
- 提供屏幕关闭后纯后台语音模式。

## 5. 验收标准（How）

所有验收标准必须可验证、可复现。

| 编号 | 验收项 | 验收标准 | 验证方式 |
|------|--------|----------|----------|
| AC01 | 基础组网 | 3 台 Android 手机在无移动网络环境下，启动应用后 60 秒内完成自动组网 | 实机测试 |
| AC02 | 直连语音 | 2 台手机在 30m 距离内可进行双向语音通话，语音延迟 < 300ms | 实机测试 + 录音测延迟 |
| AC03 | 多跳中继 | 3 台手机排成直线，A 与 C 距离 80-150m（超出 A-C 直连范围），B 位于中间，A 与 C 可稳定通话 | 实机测试 |
| AC04 | 位置共享 | 组网成功后，每个节点可在应用内看到其他节点的相对位置更新 | 实机测试 |
| AC05 | 后台运行 | 应用退至后台或锁屏后，仍能接收并播放语音 | 实机测试 |
| AC06 | 耳机支持 | 使用有线耳机与蓝牙耳机分别进行通话，麦克风与扬声器工作正常 | 实机测试 |
| AC07 | 开源合规 | 代码仓库包含 LICENSE、README、BUILD.md，可在 clean 环境按文档构建出 APK | CI/文档审查 |
| AC08 | 网络扩展性 | 5 台手机组成 3-4 跳链路时，端到端语音仍可听懂，延迟 < 1s | 实机测试 |

- **P2（Could have）**：离线文字消息、语音录音、离线地图。

## 7. 边界与排除项

### 7.1 范围内
- Android 12+ 手机端应用；
- 基于手机 Wi-Fi Direct 组网；
- 离线环境下的语音与位置数据交换；
- 开源代码与文档。

### 7.2 范围外（Won't have / 暂不实现）
- iOS 版本（首期）；
- 通过互联网/服务器中转的远程通话；
- 脱离手机硬件的独立硬件对讲机；
- 付费服务、账号系统、云端存储；
- 超出手机无线模块物理能力的长距离通信（如 5km+ 无中继直线距离）。

### 7.3 关键假设与风险

| 假设 | 风险 | 缓解措施 |
|------|------|----------|
| 用户可接受首次手动配置网络 | 配置复杂导致流失 | 提供清晰引导与默认推荐方案 |
| 手机 Wi-Fi Direct 在户外可达 100m | 实际距离受地形、电量、天线影响 | 实机测试不同场景，设置合理预期 |
| 多跳中继延迟可接受 | 跳数增加导致延迟累积 | 优化编解码与路由，限制最大跳数 |
| Android 后台限制不阻断服务 | 系统杀后台导致断连 | 引导用户设置电池白名单，使用前台服务 |

## 8. 用户故事与验收标准

### 8.1 用户故事 1：作为徒步领队，我希望在无网环境下与全队保持语音联系

**验收标准：**
- 启动应用后，所有队员手机在 60 秒内完成组网；
- 领队说话，所有队员能听到；
- 任意队员说话，领队能听到；
- 通话过程中延迟 < 300ms（单跳）或 < 1s（多跳）。

### 8.2 用户故事 2：作为队伍中间成员，我希望我的手机能自动为中继节点

**验收标准：**
- 当前后队友不在直连范围时，中间手机自动转发语音包；
- 中继过程对用户无感知，无需手动切换；
- 中间节点离开网络时，拓扑自动重新收敛。

### 8.3 用户故事 3：作为骑行队员，我希望通过蓝牙耳机免提通话

**验收标准：**
- 应用可正确识别蓝牙耳机麦克风与扬声器；
- 通话过程中可通过耳机按键切换静音或挂断；
- 蓝牙耳机断开时，自动切换回手机扬声器/麦克风。

### 8.4 用户故事 4：作为团队成员，我希望看到队友相对位置

**验收标准：**
- 组网成功后，方位界面显示每个队友的相对方位与距离；
- 位置刷新频率 ≥ 1 次/5 秒；
- 位置数据仅在本地网状网络内传输，不上云。

## 9. 跨角色转译

### 9.1 产品语言（Product）
> 做一款“无网也能群聊”的户外通讯工具。核心价值是**去中心化、不依赖运营商、覆盖前后拉开的小队**。首期聚焦 2-5 人小队，用语音+位置共享解决基本沟通与安全感知问题。

### 9.2 开发语言（Development）
> 需要实现：
> - Android 应用框架（Kotlin/Java）；
> - 本地 mesh 网络层（基于 Wi-Fi Direct / Hotspot）；
> - 语音采集/播放（AudioRecord/AudioTrack）与低码率编解码（如 Opus）；
> - 多跳路由协议（如 AODV/OLSR 简化版或自定义泛洪/贪心转发）；
> - 位置广播（GPS/北斗 + 本地广播）；
> - 开源仓库与 CI 构建。

### 9.3 测试语言（QA）
> 关键验证：
> - 实机组网稳定性（3-5 台设备）；
> - 单跳/多跳语音延迟与可懂度；
> - 后台保活与断线重连；
> - 不同耳机类型兼容性；
> - 无网环境下的位置同步；
> - 开源构建可复现性。

### 9.4 设计语言（Design）
> - 首页：一键组网/通话、节点列表、网络状态；
> - 方位页：队友相对方位与距离；
> - 设置页：组网方式、音频设备、省电模式、电池白名单引导；
> - 首次启动引导：权限申请、组网说明、耳机配对提示。

### 9.5 PM 语言（Project Management）
> - 项目周期：3-6 个月；
> - 关键路径：组网方案选型 → 直连语音 Demo → 多跳中继 → 位置共享 → 后台优化 → 开源发布；
> - 外部依赖：Android 权限策略、设备兼容性、户外测试场地；
> - 建议先交付 MVP 验证核心链路，再迭代增强功能。
