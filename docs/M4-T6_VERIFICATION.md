# M4-T6 真机功耗验证方案

> 验证目标：确认省电模式相较普通模式，15 分钟通话耗电下降 ≥ 20%。

---

## 1. 测试环境

| 项目 | 要求 |
|------|------|
| 设备 A | 一加 11（`62978a15`） |
| 设备 B | 华为/荣耀 Mate 30 Pro 5G（`2KE0219B02018194`） |
| 应用版本 | 最新 `main` 分支 debug APK（`81cce23` 及之后） |
| 网络 | 关闭移动数据与 Wi-Fi，使用 Wi-Fi Direct GO/Client 模式 |
| 亮度 | 两台设备屏幕亮度均固定为 50%，关闭自动亮度 |
| 后台 | 清理后台应用，仅保留 OffGrid |
| 电量起始 | 每轮测试起始电量 ≥ 50%，避免低电量降频影响 |

---

## 2. 测试方法

1. 设备 A 固定为 **Group Owner**，设备 B 固定为 **Client**；
2. 双方建立通话，持续 **15 分钟**；
3. 记录通话前后电量百分比；
4. 每轮测试结束后，关闭应用并静置 2 分钟，待电量稳定后记录；
5. 普通模式与省电模式各跑 **3 轮**，取平均值。

---

## 3. 测试步骤

### 普通模式

1. Settings → Power → 关闭 Power saving mode；
2. 建立通话，保持屏幕常亮；
3. 15 分钟后结束通话，记录电量消耗。

### 省电模式

1. Settings → Power → 开启 Power saving mode；
2. 建立通话，保持屏幕常亮；
3. 15 分钟后结束通话，记录电量消耗。

## 5. 验收公式

```
省电比例 = (普通模式平均耗电 - 省电模式平均耗电) / 普通模式平均耗电 × 100%
```

**验收**：省电比例 ≥ 20%。

---

## 6. 未达标时的排查方向

| 现象 | 可能原因 | 下一步 |
|------|----------|--------|
| 省电效果不足 | 屏幕仍为最大功耗来源 | 尝试降低屏幕亮度或息屏测试 |
| 省电效果不足 | Opus 复杂度降低贡献有限 | 进一步降低采样率至 8kHz 或增大帧长 |
| 省电效果不足 | 定位间隔贡献有限 | 延长定位间隔至 10s，关闭 GPS 仅使用网络定位 |
| 音质明显下降 | 12kbps 过低 | 调整至 16kbps 折中 |

---

## 7. 风险提示

- 电量百分比为非线性指标，建议每轮起始电量尽量一致；
- 环境温度、电池老化会影响结果，建议在同一场景下连续完成；
- 若差异接近 20%，可增加轮次至 5 轮以提高置信度。
