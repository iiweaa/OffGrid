# M3 Alpha 集成测试

> 测试文档 v1.0
> 目标：验证 M3 阶段双机直连语音 + 位置共享 + 后台保活的稳定性。

---

## 1. 测试目标

- 两台目标设备通过 Wi-Fi Direct 直连后，可稳定进行双向语音通话 ≥ 15 分钟。
- 位置共享功能正常：Peer Compass 正确显示队友方位与距离。
- 锁屏后 2 分钟内仍可接收语音与位置数据。
- 无崩溃、无 ANR、前台服务保持运行。

---

## 2. 测试环境

| 项目 | 内容 |
|------|------|
| App 版本 | 0.1.0（M3） |
| 设备 A | 一加 11（Android 14） |
| 设备 B | 华为/荣耀 Mate 30 Pro 5G（Android 12） |
| 网络 | Wi-Fi Direct，关闭移动数据与 Wi-Fi 上行 |
| 场景 | 室外开阔地，建议视距内 10–50m |

---

## 3. 测试前准备

1. 两台设备安装同一构建的 APK。
2. 授予所有运行时权限：麦克风、位置、附近 Wi-Fi 设备、通知、电池优化白名单。
3. 在 Wi-Fi Direct Test 中由 A 创建 Group，B 发现并连接 A。
4. 确认双方已获取 IP（192.168.49.x）。

---

## 4. 测试场景

### S1：双机直连语音稳定性

1. A 进入 **Call** 页面，点击 **Start Call**。
2. B 进入 **Call** 页面，点击 **Start Call**。
3. 双方观察邻居列表，确认出现对方 NodeId。
4. 双方轮流说话，确认双向语音清晰、延迟可接受。
5. 持续 15 分钟，期间可小幅移动设备。

**预期结果**：
- 语音不中断、无崩溃、无爆音。
- 邻居列表保持 1 个在线节点，LastSeen 持续刷新。

### S2：位置共享与罗盘

1. 在 S1 基础上，双方进入 **Peer Compass**。
2. 双方手持设备背向移动 >10m。
3. 观察罗盘标记与 Peer List 的距离、方位变化。

**预期结果**：
- Peer List 显示队友距离与方位，刷新 ≥ 1 次/5s。
- 距离误差 < 30%（依赖 GPS 精度）。
- 罗盘标记方向与实际方位一致。

### S3：锁屏保活

1. 通话与位置共享进行中，双方按电源键锁屏。
2. 等待 2 分钟（期间可继续说话）。
3. 解锁后检查 Call 页面与 Peer Compass。

**预期结果**：
- 锁屏期间仍能通过语音听到对方。
- 解锁后邻居未超时，位置数据已更新。
- 通知栏仍显示 OffGrid 前台服务。

### S4：电池优化白名单引导

1. 在首页查看电池优化提示卡片。
2. 点击 **Request battery whitelist**，进入系统设置。
3. 允许 OffGrid 忽略电池优化。

**预期结果**：
- 卡片消失或状态更新。

## 6. 已知限制

- 当前两台测试机均不支持 Wi-Fi Direct AP-STA 并发，多跳中继未启用。
- 室内或遮挡严重时 GPS 信号弱，位置精度会下降。
- 电池优化白名单设置因 OEM 而异，部分系统可能无法直接跳转。
