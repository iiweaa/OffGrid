# M5-T1 户外实测方案（已取消）

> ⚠️ **状态：已取消（2026-06-27 产品决策）**
>
> 户外实测不再执行，本文档仅作归档。下方内容保留原始方案供参考。

---

> 任务 ID：M5-T1（已取消）  
> 原负责人：`pro-general-pm` / `pro-android-system-test`  
> 原依赖：M4-T8（Beta 版本测试通过）  
> 原目标：在真实户外环境下验证 OffGrid v1.0 候选版本的语音、连接与续航表现，输出可执行的实测方案与数据记录模板。
>
> 产品决策更新：
> - 省电模式已从需求中删除，不再开发和测试；
> - 位置共享功能保留，但户外位置精度不再作为实测必测项；
> - 户外实测整体取消。

## 1. 测试目标

| 目标 | 说明 |
|------|------|
| 验证有效通信距离 | 在空旷/遮挡/起伏地形下，测定 Wi-Fi Direct 直连语音的稳定距离与极限距离 |
| 评估语音质量与延迟 | 记录不同距离与场景下的语音断续、延迟、可懂度 |
| 评估后台续航 | 锁屏/后台运行 30 分钟，记录耗电与进程存活情况 |
| 发现户外专属 Bug | 收集 P0/P1 缺陷，作为 M5-T3 修复输入 |

## 2. 测试范围

### 2.1 在测功能

- 手动 GO/Client 组网（M4-T5）
- 全双工语音通话（AudioEngine + Opus）
- 音频路由：扬声器 / 有线耳机 / 蓝牙耳机（M4-T2 / M4-T3）
- 后台保活（M3-T7 / M4-T1）

### 2.2 不在测范围

- 多跳中继（目标机型不支持 AP-STA 并发，M3 已明确降级）
- 3 台及以上设备组网
- 非目标 Android 机型兼容性
- 省电模式（已从产品需求中删除）
- 户外位置共享精度实测（位置功能保留，但不作为 M5 必测项）

## 4. 测试场地建议

| 场景 | 场地要求 | 测试目的 |
|------|----------|----------|
| 空旷平地 | 公园草坪、操场、广场，视线无遮挡，长度 ≥ 200m | 测定最大视距通信距离 |
| 林间/灌木遮挡 | 树木密度中等的步道，长度 ≥ 100m | 评估植被遮挡对信号与语音的影响 |
| 起伏地形 | 缓坡或台阶，垂直高度差 5~20m | 评估高度差对连接稳定性的影响 |
| 移动场景 | 沿步道相向/反向行走，速度 1~2m/s | 模拟徒步时的动态连接 |

> 由测试执行方根据实际情况替换为具体场地名称与坐标。

## 5. 测试用例

### 前置条件

1. 两部手机均安装同一版本 APK（基于 M4-T8 后的 `main` 分支构建）。
2. 关闭移动数据与 Wi-Fi（确认仅 Wi-Fi Direct 工作）。
3. 蓝牙已配对耳机（如测试蓝牙耳机场景）。
4. 电量 ≥ 60%，并记录初始电量。
5. 授予位置、麦克风、附近设备、电池优化白名单等权限。
6. 启动 `adb logcat` 并记录到文件（见第 8 节）。

### OC-1 空旷平地距离极限

| 项目 | 内容 |
|------|------|
| 目的 | 测定空旷环境下语音稳定通信的最大距离 |
| 步骤 | 1. A 为 GO，B 为 Client，建立连接；<br>2. 两人在同一直线上背向移动，每 25m 停顿一次进行 30 秒双向通话；<br>3. 直至一方完全听不到语音或邻居表离线。 |
| 记录 | 稳定距离、断续距离、掉线距离、建立时间 |
| 验收 | 50m 内通话清晰；100m 内可懂；极限距离 ≥ 100m 为良好 |

### OC-2 遮挡环境语音稳定性

| 项目 | 内容 |
|------|------|
| 目的 | 评估树木/灌木遮挡对信号与语音的影响 |
| 步骤 | 1. 在林间步道建立连接；<br>2. 保持 50m 距离，中间有 1~3 棵树遮挡；<br>3. 持续通话 2 分钟，记录断续次数。 |
| 记录 | 断续次数、恢复时间、主观 MOS |
| 验收 | 50m 遮挡下断续 ≤ 3 次/2min，可自动恢复 |

### OC-3 高度差影响

| 项目 | 内容 |
|------|------|
| 目的 | 评估上下坡/台阶造成的高度差影响 |
| 步骤 | 1. 水平距离 30m，垂直高度差 5m/10m/15m；<br>2. 分别测试 A 在上、B 在下与相反方向。 |
| 记录 | 每种高度差下的语音质量、邻居表状态 |
| 验收 | 高度差 ≤ 10m 时语音可懂；≤ 15m 时仍能维持连接 |

### OC-4 移动场景连续通话

| 项目 | 内容 |
|------|------|
| 目的 | 模拟徒步时动态连接稳定性 |
| 步骤 | 1. 两人相距 10m 建立连接；<br>2. 沿步道同向行走 100m，再反向走远至 100m；<br>3. 全程保持通话。 |
| 记录 | 掉线次数、重连时间、语音断续次数 |
| 验收 | 100m 往返过程中掉线 ≤ 1 次，重连 ≤ 15s |

### OC-5 后台/锁屏 30 分钟续航

| 项目 | 内容 |
|------|------|
| 目的 | 验证后台保活与续航能力 |
| 步骤 | 1. 建立连接并通话；<br>2. 双方按 Home 键退至后台，锁屏；<br>3. 保持 30 分钟，每 5 分钟通过蓝牙耳机说一句提示词；<br>4. 结束后解锁，检查邻居表与电量。 |
| 记录 | 初始/结束电量、是否掉线、是否有 ANR/Crash |
| 验收 | 30 分钟内不掉线，耗电 ≤ 15%（以 4500mAh 为参考） |

### OC-6 蓝牙耳机户外使用

| 项目 | 内容 |
|------|------|
| 目的 | 验证蓝牙耳机在户外复杂环境下的音频路由与按键控制 |
| 步骤 | 1. 双方均连接蓝牙耳机；<br>2. 完成 OC-1 中 50m 通话；<br>3. 测试短按静音、长按挂断；<br>4. 中途断开/重连蓝牙耳机，观察音频是否自动切换。 |
| 记录 | 蓝牙音频是否走耳机、按键响应、切换延迟 |
| 验收 | 蓝牙音频正确路由；按键功能符合 M4-T3 设计 |

## 6. 度量指标与记录表

### 6.1 通用指标

| 指标 | 说明 | 记录方式 |
|------|------|----------|
| 连接建立时间 | 点击「Connect」到邻居表显示对端 | logcat 时间戳 |
| 语音延迟 | 端到端感知延迟 | 主观拍手回声法 / 可选音频环回测试 |
| 语音质量 MOS | 1~5 分，5 为极佳 | 测试人员现场评分 |
| 断续次数 | 通话中可感知的卡顿/静音次数 | 人工计数 |
| 掉线次数 | 邻居表离线或完全无音 | 人工 + logcat |
| 重连时间 | 掉线后到自动恢复通话的时间 | 秒表 / logcat |
| 耗电 | 测试前后电量百分比 | `dumpsys battery` / 系统状态栏 |
| ANR/Crash | 是否出现崩溃或无响应 | logcat |

### 6.2 数据记录表模板

```markdown
## OC-X：{用例名称}

| 时间 | 场景 | 实际距离 | 语音质量 MOS | 断续 | 掉线 | 电量(A) | 电量(B) | 备注 |
|------|------|----------|--------------|------|------|---------|---------|------|
|      |      |          |              |      |      |         |         |      |
```

每个 OC 用例单独一张表，原始数据保存到 `logs/m5t2-outdoor/`。

## 8. 日志与数据采集

### 8.1 采集命令

```bash
# 每部设备单独终端
adb -s <device_id> logcat -v threadtime > logs/m5t2-outdoor/logcat-<device_id>.log 2>&1 &

# 电量快照
adb -s <device_id> shell dumpsys battery > logs/m5t2-outdoor/battery-<device_id>-<时间>.txt

adb -s <device_id> shell screencap -p > logs/m5t2-outdoor/screenshot-<device_id>-<场景>.png
```

## 9. 风险与应对

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| 户外天气突变（雨/大风） | 中 | 中 | 提前查看天气预报，准备备用室内日；小雨可改测近距离/楼道场景 |
| 设备电量不足 | 中 | 高 | 携带充电宝，每轮测试前电量 ≥ 60% |
| 场地 Wi-Fi 干扰 | 中 | 中 | 选择人少区域；若干扰严重记录 SSID 环境并改用 Channel 自动模式 |
| 安全风险（车辆/陡坡） | 低 | 高 | 选择公园步道，避免交通道路；两人保持可见范围 |
| 蓝牙耳机兼容性问题 | 中 | 中 | 准备 2 副不同品牌耳机，记录型号 |
